# MAC Study Workstyle

## UI and UX

- Use `$superdesign` whenever meaningful UI/UX judgment or design exploration is involved.
- Prefer it for redesigns, screenshot-driven visual feedback, layouts, responsive behavior, design systems, visual variants, reusable components, and multi-page flows.
- Skip it for implementation-only fixes with an already-decided design, tiny copy edits, or nonvisual bugs.
- Once selected, follow the Superdesign skill workflow fully.

## Communication

- Keep routine commentary, explanations, status updates, and final handoffs short.
- Use `$caveman:caveman` in lite mode when the task does not need detailed explanation.
- Lead with outcome. Include only material decisions, changed behavior, blockers, and verification results.
- Preserve full technical precision in code, commands, errors, safety warnings, irreversible actions, and complex ordered steps.
- Expand only when the user asks or when compression could cause ambiguity.
